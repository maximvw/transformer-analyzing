# Оригинальные эксперименты: 4x4-digit Multiplication

Справочник по экспериментам из статей:
- **"Why Can't Transformers Learn Multiplication?"** (Bai et al., 2025, arXiv:2510.00184)
- **"From Explicit CoT to Implicit CoT: Learning to Internalize CoT Step by Step"** (Deng, Choi, Shieber, 2024, arXiv:2405.14838)

---

## 1. Задача и формат данных

**Задача:** 4x4-digit multiplication (умножение двух 4-значных чисел).

**Формат входа:** Цифры записываются в порядке least-significant digit first (от младшего разряда к старшему).

Пример: `8331 * 5015` записывается как `1338 * 5105`.

**Формат полной последовательности (для ICoT):**
```
a0a1a2a3 * b0b1b2b3 %% q0...qT #### c0c1...c7
```
где:
- `a0a1a2a3`, `b0b1b2b3` — цифры операндов (least-significant first)
- `%%` — разделитель между входом и chain-of-thought
- `q0...qT` — промежуточные CoT-токены
- `####` — разделитель между CoT и ответом
- `c0c1...c7` — 8 цифр результата (least-significant first)

**Формат для SFT (без CoT):**
```
a0a1a2a3 * b0b1b2b3 %% #### c0c1...c7
```

**Пример CoT-формата для 12 x 34:**
```
12 * 34 = 48 + 360 (408) = 408
         12*4   12*30  running sum
```
Реальный формат данных (пример `8331 * 5015`):
```
1338 * 5105||5614 + 013380(569421) + 0000000(5694210) + 0005561%% #### 56997714
```

## 2. Архитектура модели

| Параметр | Значение |
|---|---|
| Базовая архитектура | GPT-2 (decoder-only) |
| Число слоёв | 2 |
| Число голов внимания | 4 |
| Размерность модели (d_model) | 256 (стандарт для GPT-2 small) |
| Общее число параметров | ~54M |
| Обучение | С нуля (from scratch), без предобучения |

**Почему 2L4H?** Авторы перебрали архитектуры и нашли, что 2-layer 4-head — это минимальная конфигурация, при которой ICoT работает, а SFT проваливается. Это позволяет чётко изолировать разницу между методами обучения.

**Масштабирование:** 12-layer 8-head модель на SFT показывает такую же ~1% accuracy, что и 2L4H. Даже 2B-параметровая модель (Yang et al., 2023) достигает лишь 95%. Проблема не в размере модели.

## 3. Эксперимент 1: ICoT (Implicit Chain-of-Thought)

### 3.1 Метод Stepwise Internalization (из статьи Deng et al., 2024)

**Идея:** Начинаем с полной цепочки CoT, и на каждой эпохе убираем фиксированное количество CoT-токенов **слева** (от начала цепочки). Модель вынуждена интернализировать ранние шаги рассуждений в скрытых состояниях.

**Формально:**
- На шаге t удаляется s(t) токенов: `s(t) = floor(Delta * t / T)`
- `T` — число шагов в эпохе, `Delta` — число CoT-токенов, удаляемых за эпоху
- Когда s(t) >= общего числа CoT-токенов, все удалены → модель делает implicit CoT

**Стадии обучения (пример для 12 x 34):**
```
Stage 0: 2 1 x 4 3 = | 8 4 + 0 6 3 | = 8 0 4   (полный CoT)
Stage 1: 2 1 x 4 3 = |   4 + 0 6 3 | = 8 0 4   (убран 1 токен)
Stage 2: 2 1 x 4 3 = |     + 0 6 3 | = 8 0 4
...
Stage 6: 2 1 x 4 3 = |             | = 8 0 4   (implicit CoT)
```

### 3.2 Гиперпараметры ICoT-обучения (из статьи Bai et al.)

| Параметр | Значение |
|---|---|
| Размер обучающей выборки | 80,800 примеров |
| Размер validation/test | 1,000 / 1,000 |
| Learning rate | 5e-5 |
| Optimizer | AdamW |
| Delta (CoT-токенов за эпоху) | 8 |
| Число эпох до сходимости | 13 |
| Итоговая accuracy | **100%** на test set |

### 3.3 Стабилизация обучения

Два ключевых трюка для стабильности Stepwise Internalization:

1. **Optimizer Reset:** При удалении очередного CoT-токена сбрасывается состояние оптимизатора (AdamW). Без этого второй момент AdamW содержит устаревшие оценки, что дестабилизирует обучение.

2. **Removal Smoothing:** К числу удаляемых токенов добавляется случайный offset `o ~ Exp(lambda)`, `lambda = 4`. В 98% случаев o=0, в ~2% — удаляется на 1+ токен больше. Это сглаживает переход между стадиями.

### 3.4 Ablation Studies (Deng et al.)

- **Без Removal Smoothing:** accuracy падает до 0 после ~50 шагов и не восстанавливается
- **Без Optimizer Reset:** accuracy падает около 100 шагов и не восстанавливается
- **Right-Side Removal** (удаление справа): значительно хуже, т.к. токены в конце CoT зависят от предыдущих
- **Aggressive Removal** (Delta=16 вместо 8): обучение не сходится

## 4. Эксперимент 2: SFT (Standard Fine-Tuning)

### 4.1 Настройка

| Параметр | Значение |
|---|---|
| Архитектура | Та же 2L4H GPT-2 |
| Формат данных | `a0a1a2a3 * b0b1b2b3 %% #### c0...c7` (без CoT) |
| Learning rate | 5e-5 |
| Число эпох | 13 (loss/accuracy плато) |
| Итоговая accuracy (exact match) | **< 1%** |
| Digit-level accuracy | ~81% |
| При масштабировании (12L8H) | Те же < 1% exact, ~80% digit-level |

### 4.2 Динамика обучения (ключевое наблюдение)

Авторы отслеживали **per-token loss** и **per-token gradient norms** на протяжении обучения (Figure 7a в статье):

1. **Первыми выучиваются c0, c1** (младшие цифры) — их loss падает до нуля первым. Это логично: c0 = (a0*b0) mod 10 зависит только от одной пары цифр.

2. **Затем выучивается c7** (старшая цифра) — она тоже относительно проста (зависит от carry от c6, но carry обычно мал).

3. **Модель затем выучивает c2**, но **c3-c6 никогда не выучиваются**. Loss для средних цифр выходит на плато и не падает.

4. **Gradient norms:** после того как c0, c1, c7 выучены, только средние цифры c3-c6 генерируют ненулевой gradient. Но этого градиента недостаточно, чтобы модель выучила правильные long-range dependencies.

### 4.3 Почему SFT проваливается: отсутствие long-range dependencies

Для вычисления цифры `c_k` результата нужна промежуточная величина:
```
s_k = sum(a_i * b_j) для всех i+j=k    (сумма partial products)
c_hat_k = s_k + r_{k-1}                  (running sum, с учётом carry)
c_k = c_hat_k mod 10                     (цифра результата)
r_k = floor(c_hat_k / 10)                (carry для следующей цифры)
```

Для средних цифр (k=3..6) `s_k` зависит от **многих** пар `(a_i, b_j)`, и carry `r_{k-1}` зависит от **всей** предшествующей вычислительной цепочки. Модель SFT не формирует эти long-range dependencies при стандартном autoregressive обучении — gradient descent застревает в локальном оптимуме.

## 5. Эксперимент 3: Auxiliary Loss (обучение без ICoT)

### 5.1 Идея

Вместо ICoT используется **auxiliary loss**, который напрямую побуждает модель формировать правильные long-range dependencies. К выходам attention heads второго слоя подключаются линейные regression heads, которые предсказывают промежуточную величину `c_hat_k` (running sum).

### 5.2 Формулировка loss

**Linear regression probe:** Для каждой головы внимания h во втором слое обучается вектор `w_h`:
```
z_i^h = w_h^T * ATT_h^2(·)
```

**Auxiliary loss (MSE):**
```
L_aux = (1/H) * sum_h (1/8) * sum_{i=0}^{7} (z_i^h - c_hat_i)^2
```
где `c_hat_i` — истинное значение running sum на позиции `t_{c_i}`.

**Полный loss:**
```
L = L_LM + lambda * L_aux
```
где `L_LM` — стандартный autoregressive (cross-entropy) loss.

### 5.3 Куда крепится probe

- Probe крепится к **выходу attention heads второго (последнего) слоя**, до MLP
- Конкретно: `ATT_h^2(·)` — выход head h после attention, но **до** прохода через MLP
- Probe обучается на всех H=4 головах (общий loss усредняется по головам)
- Предсказание делается на каждом timestep `t_{c_k}` (позиция, где модель генерирует `c_k`)

### 5.4 Таргет для probe

Таргет — **running sum** `c_hat_k`:
```
c_hat_k = s_k + r_{k-1}
```
где:
- `s_k = sum(a_i * b_j | i+j=k)` — сумма partial products
- `r_{k-1}` — carry от предыдущего шага
- `c_hat_0 = s_0 = a_0 * b_0`
- `c_hat_1 = s_1 + floor(c_hat_0 / 10)`
- и т.д.

Значения `c_hat_k` для 4x4 умножения лежат примерно в диапазоне [0, ~250].

### 5.5 Гиперпараметры

| Параметр | Значение |
|---|---|
| Архитектура | Та же 2L4H GPT-2 |
| Формат данных | SFT-формат (без CoT) |
| lambda (вес auxiliary loss) | Не указан точно, подбирался |
| Число шагов до сходимости | ~1M gradient steps |
| Итоговая accuracy | **99%** |

### 5.6 Динамика обучения (Figure 7b)

С auxiliary loss динамика **принципиально отличается** от SFT:

1. Сначала выучиваются крайние цифры: c0, c1, c7
2. Затем модель учится **от краёв к центру**: c2, c6, затем c3, c4, c5
3. В отличие от SFT, loss для средних цифр **не** выходит на плато — вместо этого они постепенно выучиваются
4. Это прямое следствие того, что auxiliary loss "заставляет" модель формировать long-range dependencies через supervision на running sum

## 6. Анализ внутренних состояний трансформера

### 6.1 Logit Attribution

**Метод:** Для каждого примера `a0a1a2a3 * b0b1b2b3` заменяем одну цифру на случайную (counterfactual) и измеряем изменение логитов:
```
Delta_{t,k} = logit_{c_k}(ORIG) - logit_{c_k}(COUNTER_t)
```
Это показывает, какие входные цифры `a_i, b_j` влияют на предсказание `c_k`.

**Результаты (Figure 2):**
- **ICoT модель:** Правильная структура зависимостей — для c_k наибольший эффект имеют пары `{a_i*b_j | i+j=k}`, с убывающим влиянием более ранних пар
- **SFT модель:** Модель видит зависимости только для крайних цифр (c0, c1, c7), но **не** видит правильных зависимостей для средних цифр

### 6.2 Linear Probing для c_hat_k (running sum)

**Метод:** На скрытых состояниях `h^{2,mid}` (слой 2, после attention, до MLP) обучается линейный probe `w_k`, предсказывающий `c_hat_k`:
```
w_k * h^{2,mid}_{t_{c_k}} ≈ c_hat_k     (MSE loss)
```

**Результаты (Figure 3):**

| Цифра | ICoT MAE | SFT MAE |
|---|---|---|
| c_hat_2 | 2.00 | 93.69 |
| c_hat_3 | 1.89 | 113.27 |
| c_hat_4 | 1.74 | 74.47 |
| c_hat_5 | 0.97 | 79.40 |
| c_hat_6 | 0.56 | 28.22 |

ICoT модель имеет MAE < 2 для всех цифр — running sum **линейно декодируем** из скрытых состояний. SFT модель — MAE от 28 до 113, промежуточная информация **не** представлена в скрытых состояниях.

### 6.3 Attention Trees (механизм ICoT)

**Наблюдение:** ICoT модель организует attention в виде **binary-tree-like directed acyclic graph**.

**Слой 1:** Каждая голова attention обращает внимание на **пару** цифр `(a_i, b_j)`. Это позволяет:
- Вычислять partial product `a_i * b_j`
- **Кэшировать** результат в скрытом состоянии `h^1_t` на позиции t

**Слой 2:** На позиции `t_{c_k}` головы attention второго слоя обращаются к позициям, где были закэшированы нужные partial products `{a_i*b_j | i+j ≤ k}`.

**Пример для c2:** Для предсказания `c2` нужны `a2b0, a1b1, a0b2` и carry `c_hat_1`. Layer 2 heads обращаются к позициям `(b0, b2, c1, c0)`, где Layer 1 закэшировал соответствующие partial products.

### 6.4 Feature Geometry

**Minkowski Sums:** Выходы attention первого слоя, обращающиеся к двум цифрам (a_i, b_j), формируют **суммы Минковского** в пространстве представлений:
```
ATT^1(i,j) = alpha * A_i + (1-alpha) * B_j + epsilon
```
Это создаёт вложенные кластеры: глобальная структура по одной цифре, локальная — по другой.

**Pentagonal Prism (Fourier базис):**
- Embeddings E, выходные веса MLP, финальные hidden states кодируют цифры через **Fourier базис**
- Используются частоты k ∈ {0, 1, 2, 5}: `Phi(n) = [1(n), cos(2pi*n/10), sin(2pi*n/10), cos(2pi*n/5), sin(2pi*n/5), p(n)]`
- В 3D PCA: цифры 0-9 образуют **пятиугольную призму** (pentagonal prism)
- PC1 = parity (чёт/нечет), PC2-PC3 = k=2 Fourier pair → два пятиугольника (чётные и нечётные цифры)
- Медианный R^2 Fourier fit: 0.84-0.99 для 6D базиса, 1.0 для 8D

### 6.5 Auxiliary Loss модель: тот же механизм?

**Ответ: да, похожий, но не идентичный.**

Модель, обученная с auxiliary loss, также формирует:
- **Binary attention tree** (sparse attention в Layer 1 к парам цифр — красные рамки на Figure 8)
- **Дополнительно:** одна из голов Layer 2 (Head 2) формирует "parallelogram" pattern — одновременно обращается ко **всем** нужным цифрам `{a_i, b_i | i ≤ k}`, что напоминает logit attribution pattern (Figure 2)

## 8. Ключевые выводы для переноса на другие задачи

1. **Long-range dependencies** — корневая проблема. Если задача требует агрегации информации из далёких позиций входа, SFT застревает в локальном оптимуме.

2. **Auxiliary loss** — эффективный inductive bias. Достаточно supervisить промежуточные вычисления (running sum), и модель сама формирует правильную структуру.

3. **Probe крепится к attention outputs**, не к MLP — это ключевое архитектурное решение.

4. **Интерпретируемость:** Успешные модели формируют разреженные, интерпретируемые attention patterns (деревья), которые напрямую соответствуют алгоритму задачи.

5. **Масштабирование не спасает** от плохого inductive bias — 12-layer модель и даже 2B-параметровая модель не могут решить задачу через SFT.
